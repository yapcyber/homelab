# From a collection of services to an operated platform

This homelab is a hands-on engineering environment for infrastructure,
automation and defensive security. It runs a clustered virtualisation layer,
segmented networks and self-hosted services, with remote access restricted to a
private VPN.

The interesting part is not the application count. It is the operating model:
configuration is versioned, changes are validated, exposed container ports are
filtered, failures are observable, and recovery constraints are documented.

## Current capabilities

- Infrastructure provisioning with Packer, OpenTofu and Ansible.
- Reverse proxy, identity, monitoring, logs and vulnerability management.
- Host and network security telemetry, plus custom detection content.
- Daily application-aware backups with encrypted sensitive archives.
- Automated health, certificate, drift and update checks.

## Evidence, not a perfect-lab narrative

The project records failed assumptions and remaining risks. Local backups are
not yet an off-site copy. Some application migrations still require a manual,
staged deployment. Security appliances deliberately keep privileged update
commands behind interactive administration. These boundaries are treated as
engineering inputs rather than hidden.

See the [engineering case studies](./case-studies.md) for examples of diagnosis,
remediation and lessons learned.
